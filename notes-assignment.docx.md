**Тестовое задание: Модуль «Заметки»**

*Внутренняя система заметок для сотрудников*

# **1\. Модель данных**

Создать SQL-миграцию с таблицей notes:

| Поле | Тип | Описание |
| :---- | :---- | :---- |
| id | INT, PK, AUTO\_INCREMENT | Идентификатор |
| user\_id | INT, NOT NULL | Автор заметки (FK → users.id) |
| title | VARCHAR(255), NOT NULL | Заголовок |
| content | TEXT | Содержимое заметки |
| color | VARCHAR(7), DEFAULT '\#6366f1' | Цвет-метка (hex) |
| is\_pinned | TINYINT(1), DEFAULT 0 | Закреплена ли заметка |
| created\_at | DATETIME, DEFAULT CURRENT\_TIMESTAMP | Дата создания |
| updated\_at | DATETIME, DEFAULT CURRENT\_TIMESTAMP ON UPDATE | Дата обновления |

# **2\. Backend**

## **2.1 Контроллер (полный CRUD)**

| Метод | Описание |
| :---- | :---- |
| index | Список заметок текущего пользователя с пагинацией (20 на страницу). Поиск по заголовку (GET-параметр q). Закрепленные заметки — всегда сверху. |
| create | Форма создания заметки |
| store | Сохранение новой заметки. Валидация: title обязателен, максимум 255 символов; color — валидный hex. Редирект на список с flash-сообщением. |
| edit | Форма редактирования (только свои заметки) |
| update | Обновление заметки (только свои) |
| destroy | Удаление заметки (только свои) |
| togglePin | AJAX-endpoint: переключение is\_pinned. Ответ в JSON. |

## **2.2 Требования к безопасности**

* Пользователь может управлять только своими заметками (проверка user\_id)

* Все пользовательские данные экранируются при выводе

* Prepared statements для всех запросов к БД

* CSRF-токен в формах

# **3\. Frontend**

## **3.1 Список заметок**

* Карточки заметок в сетке (grid), а не таблица

* Цветная полоска сверху карточки соответствует color заметки

* Иконка булавки для закрепленных заметок

* Кнопка pin/unpin через Alpine.js \+ fetch (без перезагрузки страницы)

* Поисковая строка с фильтрацией

* Пагинация

## **3.2 Форма создания / редактирования**

* Поля: заголовок, содержимое (textarea), выбор цвета

* Набор из 6 предустановленных цветов в виде кружков, выбор через Alpine.js

* Валидационные ошибки выводятся под соответствующими полями

* Кнопки «Сохранить» и «Отмена»

# **4\. Маршруты**

Все маршруты должны требовать авторизацию.

| Метод | Путь | Действие |
| :---- | :---- | :---- |
| GET | /notes | Список заметок |
| GET | /notes/create | Форма создания |
| POST | /notes | Сохранение |
| GET | /notes/{id}/edit | Форма редактирования |
| POST | /notes/{id} | Обновление |
| POST | /notes/{id}/delete | Удаление |
| POST | /notes/{id}/toggle-pin | Переключение pin |

# **5\. Общие требования**

* Срок: 3 календарных дня

* Формат: проект в Git-репозитории, merge requests всех веток и процесса разработки, не одним литым коммитом

* Обязательно: краткое описание в MR — что сделано, что не успели, какие решения приняли и почему

* В коде не должно быть закомментированных блоков, отладочного вывода (var\_dump, print\_r) и TODO без пояснений

* Допускается использование различных популярных фреймворков
